# 护栏手册 —— 无人看管的循环，靠护栏不靠运气

> 愚公第六步「交令」必须把这些焊进命令。一个跑飞的 loop 能烧光预算、删掉目录、合并坏代码——护栏不是可选项。

## 1. 成本护栏
- **迭代上限**：单元最多 N 轮没满足 done_when 就停下报告，防无限循环。
- **token 预算**：设上限，花到就停。多单元并行尤其容易爆。
- **首跑 dry-run**：第一次只输出「打算对每个单元做什么」，不真改；人确认计划再放真跑。
- **小样本起步**：先放 1～2 个单元验闭环，再放全量。

## 2. 安全护栏（人类把关）
- **高风险动作进 hard_stops**：merge 默认分支、打 tag/发版、删除文件或逻辑、外部 API 写操作、force-push——一律 blocked 交人类，永不进自动判据。
- **授权要祈使句**：用户的疑问句（「行了吧？」）是问状态不是授权；执行授权必须是祈使句（「merge 吧」「发版」）。一次授权只覆盖当次动作。

## 3. 工位护栏（继承实战教训）
- **commit 即 push**：每个过验证的提交立刻推送，不囤本地提交（后台进程失败清理曾删掉未推送提交）。
- **长任务不进后台**：clone 大仓库、跑流水线前台等完；已转后台的任务，其目录在任务终结前不复用。
- **worktree 隔离**：并行单元各自独立工作区，避免互相踩。
- **主流程心跳**：后台子 agent 产出文件长时间不增长 = 疑似卡死（多半卡在不可见的权限弹窗），叫停、捞回线索、换前台/CLI。

## 4. 工具护栏
- **优先 `gh`/`curl` CLI**，慎用 `WebFetch`/`WebSearch`/部分 MCP——后者会触发子 agent 里用户看不见的权限弹窗，导致静默挂起。一种工具连续失败立刻换 CLI，不原地重试。

## 5. 可理解性护栏
- **checker 必须用人话总结这轮改了什么**，防 comprehension debt（循环跑完没人看得懂产物变成了什么）。

## 一句话清单（贴进 /goal 护栏段）
> 高风险动作 blocked 交人类；maker 与 checker 分两个 agent；commit 即 push；
> 优先 gh/curl 不用 WebFetch；迭代上限 N；超 token 预算停；首跑 dry-run + 小样本。
