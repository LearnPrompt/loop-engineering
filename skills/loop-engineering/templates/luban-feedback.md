# luban-feedback.md —— 鲁班用起来的卡点（鲁班自举的原料）

> 愚公循环跑鲁班升级别的 skill 时，把「鲁班这工具本身哪卡了/缺什么手艺」结构化攒在这里。
> 攒够阈值（默认 5 条）就触发「用鲁班打磨鲁班自己」的元 goal。

## 反馈条目格式

```markdown
### F-001
- 现象：<鲁班在做什么时出了什么问题 / 缺了什么>
- 触发步骤：<在哪个 skill、五步中的哪一步、什么输入下触发>
- 影响：<导致升级质量/效率受了什么损>
- 建议：<鲁班该怎么改：加一条规矩 / 一个工具 / 一段 prompt>
- 来源：<skill 仓库 + PR/commit 链接，可查证>
- 状态：open | 已进 self-upgrade PR | wontfix（附理由）
```
